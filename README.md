# pcf-syntax-highlighter

> A PowerApps PCF control for VS Code–quality syntax highlighting powered by [Shiki](https://shiki.matsu.io) — supporting 140+ languages and 25+ themes.

## Overview

`pcf-syntax-highlighter` brings VS Code–quality code rendering into **Power Apps** and **Power Pages** as a drop-in PCF (PowerApps Component Framework) control. It uses the same [Shiki](https://github.com/shikijs/shiki) engine that powers VS Code and was popularised in the WordPress ecosystem by [code-block-pro](https://github.com/KevinBatdorf/code-block-pro).

Bind any text field containing source code and get beautiful, accessible, themed output — without leaving the Power Platform.

---

## Features

- 🎨 **25+ built-in VS Code themes** — `github-dark`, `nord`, `one-dark-pro`, `dracula`, `tokyo-night`, and more
- 💬 **140+ programming languages** — TypeScript, Python, SQL, C#, PowerFx, JSON, YAML, and more
- 🔢 **Optional line numbers**
- 📋 **Copy-to-clipboard button** with animated feedback
- 🏷️ **Configurable header label** — display file names or language badges
- 🔡 **Configurable font size**
- ⚡ **Highlighter instance caching** — themes and languages loaded once per page session
- ♿ **Accessible** — proper `aria-label` on interactive elements
- 🔌 **No external runtime dependencies** — Shiki is bundled at build time

## Properties

| Property | Type | Default | Description |
|---|---|---|---|
| `code` | Multiple (bound) | — | **Required.** The source code string to render. Bind to a text column. |
| `language` | SingleLine.Text | `typescript` | Programming language for syntax highlighting. See [supported languages](https://shiki.matsu.io/languages). |
| `theme` | SingleLine.Text | `github-dark` | VS Code / Shiki theme name. See [supported themes](https://shiki.matsu.io/themes). |
| `showLineNumbers` | TwoOptions | `true` | Show or hide line numbers. |
| `showCopyButton` | TwoOptions | `true` | Show or hide the copy-to-clipboard button. |
| `headerLabel` | SingleLine.Text | `""` | Optional header text (e.g. `index.ts`). Falls back to the language name when empty. |
| `fontSize` | Whole.None | `14` | Font size in pixels. |

## Getting Started

### Prerequisites

- [Node.js](https://nodejs.org/) v18+
- [Power Platform CLI](https://learn.microsoft.com/en-us/power-platform/developer/cli/introduction) (`pac`)
- A Power Apps environment with PCF controls enabled

### Install dependencies

```bash
npm install
```

### Run in the test harness

```bash
npm start watch
```

This opens the PCF test harness at `http://localhost:8181` where you can paste code and tweak properties live.

### Build for production

```bash
npm run build
```

### Package and deploy

```bash
# Create the solution zip
pac solution init --publisher-name YourPublisher --publisher-prefix pub
pac solution add-reference --path .

# Build the solution
msbuild /t:build /restore

# Import into your environment
pac solution import --path ./bin/Debug/YourSolution.zip
```

## Development Notes

### Highlighter caching

Shiki is initialised once at module level and shared across all control instances on the page. Themes and languages are loaded lazily on first use, so only what you actually configure is downloaded.

### Fallback behaviour

If Shiki fails to load a language or theme (e.g. a typo in the property value), the control falls back gracefully to a plain `<pre><code>` block and logs the error to the browser console.

### Fonts

The control uses a monospace font stack that prefers coding fonts already installed on the user's machine:

```
'Cascadia Code', 'Fira Code', 'JetBrains Mono', 'Consolas', monospace
```

No web fonts are loaded, keeping the control fast and offline-capable.
